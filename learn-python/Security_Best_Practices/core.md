# Core Python Concepts

- Apply least privilege and avoid exposing more capability than a task requires.
- Validate, sanitize, and encode untrusted input before using it in sensitive contexts.
- Manage secrets through environment variables or secret stores, not source files.
- Use rate limiting, dependency scanning, and secure defaults to reduce common risk.
- Prefer well-maintained libraries for cryptography, auth, and validation.
- Treat security reviews as part of regular engineering, not a final checklist.
