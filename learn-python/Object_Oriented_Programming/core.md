# Core Python Concepts

- Model behavior with classes, instances, attributes, and methods.
- Use encapsulation to keep object state valid and easy to reason about.
- Choose composition or inheritance based on the relationship between objects.
- Use polymorphism so code depends on interfaces rather than concrete implementations.
- Implement dunder methods when objects should behave like built-in types.
- Keep classes focused so object design stays maintainable as the system grows.
