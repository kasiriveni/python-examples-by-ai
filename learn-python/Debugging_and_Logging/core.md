# Core Python Concepts

- Read tracebacks carefully to find the real source of a failure.
- Use `print`, assertions, breakpoints, or `pdb` to inspect runtime state.
- Log useful context with clear levels such as `DEBUG`, `INFO`, `WARNING`, and `ERROR`.
- Avoid swallowing exceptions without recording enough detail to diagnose them.
- Reproduce bugs with minimal examples before changing code.
- Profile or time sections of code when a problem might be performance-related.
