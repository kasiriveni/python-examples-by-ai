# Core Python Concepts

- Prefer readable code, descriptive names, and small focused functions.
- Follow PEP 8 style conventions and keep formatting consistent.
- Add docstrings and type hints where they improve clarity and maintenance.
- Separate business logic from I/O, framework code, and configuration.
- Raise specific exceptions, validate input, and log actionable information.
- Treat tests, dependency pinning, and environment isolation as part of normal development.
