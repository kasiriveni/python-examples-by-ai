# Core Python Concepts

- Measure before optimizing by using profilers, benchmarks, and realistic workloads.
- Think in terms of algorithmic complexity before micro-optimizing syntax.
- Reduce unnecessary allocations and repeated work in hot paths.
- Use vectorized libraries or caching when they solve the real bottleneck.
- Understand when I/O latency, CPU time, or memory pressure is the limiting factor.
- Keep optimized code maintainable unless the performance gain is truly necessary.
